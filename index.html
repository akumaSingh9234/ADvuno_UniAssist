Pre-Meeting Checklist

Meeting Flow

Section 1: Opening 5 min

Say: "Thank you for taking the time today. We have completed a thorough analysis of your current application and have prepared a comprehensive proposal for the rebuild. We will first walk through what we found in the current system, then present our recommended solution, and finally discuss timeline and next steps."

Section 2: Current Application Problems 15-20 min

Issue What to Point Out Business Impact
Performance Slow page loads (3-5 seconds), laggy authentication, user state lost when page refreshes Users abandon site, poor first impression, frustration with forms
Architecture Streamlit creates 12 separate Python processes, no data sharing between modules High server costs, difficult maintenance, inconsistent user experience
User Experience Inconsistent styling, poor mobile layout, forms lose data on navigation, excessive emojis Users abandon complex tasks, 40% lower completion rates
Security Authentication token visible in browser URL, weak session management Potential data exposure, trust issues with security-conscious users
Scalability Each user = one Python process, cannot handle more than ~50 concurrent users efficiently System will crash or slow down significantly during peak usage
Important: Be diplomatic. Never say "this is broken" or "this is bad." Instead say:
  • "There is significant opportunity to improve here..."
  • "The current approach works but has limitations at scale..."
  • "Users currently experience some friction that we can eliminate..."

Section 3: Proposed Solution 20-25 min

A. Architecture: Modular Monolith

Explain: "Instead of 12 separate applications, we build one unified system with clear internal modules. Think of it like a well-organized office building versus 12 separate buildings across the city."
What It Means Benefit for You
Single codebase with clear module boundaries Easier to develop, test, and debug
One deployment instead of 12 separate apps 60-70% lower hosting costs
Shared authentication and user data User fills profile once, available everywhere
Right-sized for a 3-person team Faster development, less complexity
Can be split into microservices later Future-proof as you grow

B. Embeddings: Self-Hosted Model

Explain: "For converting documents into searchable format, we recommend hosting our own model. This gives predictable costs regardless of how many documents students upload."
Option Monthly Cost Pros/Cons
Self-Hosted (Recommended) 2,700 - 5,000 INR fixed Fast processing, complete data privacy, predictable billing
Cloud API (OpenRouter/OpenAI) Can exceed 80,000 INR at scale Easier setup, but cost grows with every document uploaded

C. Vector Database: Qdrant

Explain: "This is a specialized database for storing and searching the document embeddings. Qdrant gives us the same capabilities as expensive alternatives at a fraction of the cost."
  • 3-5x cheaper than Pinecone for identical features
  • Advanced filtering - search by student, course, date, any criteria
  • Hybrid search - combines text matching with AI similarity
  • Can self-host for development, use managed cloud for production

D. LLM Strategy: OpenRouter Multi-Model

Explain: "Instead of being locked to one AI provider, we use a routing service that gives access to 100+ models through one connection. This means reliability, cost optimization, and flexibility."
Task Type Model Used Why
Quick Q&A, simple queries Llama 3.1 / DeepSeek (cheap) Fast, low cost, good enough for simple tasks
Regular conversations GPT-4o-mini / Gemini Good balance of quality and cost
Essay writing, SOPs, LORs Claude 3.5 Sonnet Best at long-form writing
Complex analysis, reasoning GPT-4o / Claude 3.5 Most capable for difficult questions

Premium Feature: Let users choose their AI tier (Standard free, Premium paid) = additional revenue stream.

E. Live Data: Search API Integration

Explain: "For information that changes frequently - visa rules, university rankings, scholarship deadlines - we pull fresh data from the internet in real-time instead of maintaining outdated databases."
  • Always current information (no stale data)
  • Answers include source links (builds user trust)
  • No need to manually update databases
  • Covers: visa regulations, university info, job market, scholarships

F. Custom AI Agents Per Module

Explain: "Each module gets its own specialized AI personality - a visa expert for visa questions, an essay coach for writing, a career counselor for job guidance. Each trained on curated, verified content for that domain."
Module AI Personality Trained On
Visa Ready Visa Consultant Immigration regulations, embassy guidelines
Essay & Doc Writing Coach Successful application essays, SOP examples
AI Test Prep Exam Expert JEE/NEET/SAT patterns, strategies
Wellness Support Companion Mental health best practices, coping strategies

Section 6: Q&A 15-20 min

If They Ask Your Response
Technical question you know well Answer directly and confidently. Offer to include more detail in the written proposal.
Question you are not sure about "That is a good question. I will research this thoroughly and include the answer in our detailed proposal."
About exact development costs Give high-level ranges only. Say the detailed breakdown will be in the written proposal.
Express concerns or hesitation Listen fully without interrupting. Acknowledge the concern is valid. Explain how our approach addresses it. If it cannot be fully addressed, be honest about tradeoffs.
About timeline guarantees "The 12-16 week estimate assumes the scope we discussed. We will finalize exact timelines after confirming MVP requirements."
Closing: "Thank you for your time today. To summarize our next steps: We will send you the detailed proposal and requirements form by [DATE]. Once you complete the requirements form, we will schedule a follow-up call to finalize scope and timeline. Does that work for you?"

Current UI - Problems to Demonstrate

1. Chat Interface

Point out these issues:
  • Token exposed in URL bar (see the long string after "token=") - security risk
  • Basic Streamlit default styling - looks like a prototype, not production
  • Chat history not reliably saved - users lose conversations
  • No streaming - response appears all at once after delay

2. PPT Generator

Point out these issues:
  • No progress indicator during generation - users do not know if it is working
  • Red button color suggests error or danger, confusing for primary action
  • Entire UI freezes while processing - cannot do anything else
  • Simple text input for complex task - no guidance or templates

3. Resume Builder (Essay & Doc)

Point out these issues:
  • No live preview - users cannot see what resume will look like
  • Mixed styling throughout the form - looks inconsistent
  • Form data not saved if user navigates away - work lost
  • Generic file upload component - no drag and drop feedback

4. Money Matters / Scholarships

Point out these issues:
  • "CONNECTING" status stuck in top right - shows reliability problems
  • Form is very dense with no visual breathing room - overwhelming
  • No validation feedback on inputs - users do not know if data is correct
  • Tiny stepper controls (+/-) - hard to use on mobile
